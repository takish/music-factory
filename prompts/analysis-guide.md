# 楽曲分析ガイド

このガイドは、楽曲を「設計視点」で分析し、再利用可能な形式で記録するためのものです。

## 分析の目的

1. **なぜその曲が成立しているか**を言語化する
2. **どの構造・和声・言葉の使い方が効いているか**を特定する
3. **AIに伝えると再現性が上がる要素**を抽出する

→ 単なる模倣ではなく、**設計思想の資産化**

## 著作権への配慮

分析時は以下を厳守してください：

1. **歌詞の直接引用禁止** - 歌詞の具体的なフレーズをコピーしない
2. **コード進行はローマ数字** - 機能和声として抽象化（i, IV, V, vi など）
3. **メロディ採譜なし** - 音程やリズムの具体的な記譜はしない
4. **構造・設計意図の記述** - 「なぜそう作られているか」を分析

## 良い分析の例

### 曲の本質（良い例）
```markdown
## 曲の本質
- **二重構造**: 表層（完璧なアイドル像）/ 内面（虚構・孤独）
- **最大のフック**: 構造そのもの（明るさと不穏さの同居）
- **感情設計**: 語るが、救わない
```

### コード進行（良い例）
```markdown
### Chorus
- **パターン**: i – VI – III – VII（Verseと同型）
- **設計意図**: コードは変えず意味だけを反転 →「変わらない自分」を暗示
```

### 楽器構成（良い例）
```markdown
### 楽器構成
| パート | 楽器 | 役割・特徴 |
|--------|------|-----------|
| 中心 | ピアノ + シンセ | メロディのドライバー、冷たさと輝き |
| リズム | 打ち込みドラム | 16ビート、タイト、前のめり |
| ベース | シンセベース | グルーヴィー、低域で支える |
| 装飾 | ストリングス（シンセ） | サビで広がり、感情の増幅 |
| 効果 | ボーカルチョップ、シンセスタブ | Drop・Instrumental で使用 |
```

---

## 分析のコツ

1. **「なぜ」を常に問う** - 単なる観察ではなく設計意図を推測
2. **再利用可能な形で記述** - 他の曲にも適用できる洞察
3. **対比で特徴を際立たせる** - 「〜ではなく〜」という形式
4. **感情のアークを意識** - 曲全体で感情がどう動くか
