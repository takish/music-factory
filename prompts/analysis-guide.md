# 楽曲分析ガイド

このガイドは、楽曲を「設計視点」で分析し、再利用可能な形式で記録するための正式なフォーマット定義です。

## 分析の目的

1. **なぜその曲が成立しているか**を言語化する
2. **どの構造・和声・言葉の使い方が効いているか**を特定する
3. **AIに伝えると再現性が上がる要素**を抽出する

→ 単なる模倣ではなく、**設計思想の資産化**

---

## 分析の正確性を担保する

### 情報源の優先順位

1. **実際に曲を聴く** - YouTubeリンクがあれば参照
2. **既存のコード進行分析を参照** - 楽譜サイト、分析ブログ等
3. **LLMの知識** - 上記がない場合の補助

### 不確かな情報の扱い

確信が持てない場合は「（推定）」を明記する：

```markdown
- **パターン**: i – VI – III – VII（推定）
- **BPM**: 130程度（推定）
```

### やってはいけないこと

- 歌詞の直接引用（著作権）
- メロディの採譜（著作権）
- 推測を事実のように書く
- 一般論で済ませる（「よくあるJ-Pop」等）

## 良い分析の例

### 曲の本質

```markdown
## 曲の本質
- **二重構造**: 表層（完璧なアイドル像）/ 内面（虚構・孤独）
- **最大のフック**: 構造そのもの（明るさと不穏さの同居）
- **感情設計**: 語るが、救わない
```

→ 「キャッチーなポップソング」ではなく、**なぜ刺さるか**を言語化

### コード進行

```markdown
### Chorus
- **パターン**: i – VI – III – VII（Verseと同型）
- **設計意図**: コードは変えず意味だけを反転 →「変わらない自分」を暗示
```

→ コードを書くだけでなく、**なぜそのコードなのか**を説明

### 楽器構成

```markdown
### 楽器構成
| パート | 楽器 | 役割・特徴 |
|--------|------|-----------|
| 中心 | ピアノ + シンセ | メロディのドライバー、冷たさと輝き |
| リズム | 打ち込みドラム | 16ビート + トラップ系ハイハットロール |
| ベース | 808ベース | Trap由来のスライドベース、低域の圧 |
| 装飾 | ストリングス（シンセ） | サビで広がり、感情の増幅 |
| 効果 | ボーカルチョップ、シンセスタブ | Drop・Instrumental で使用 |
```

→ 具体的な楽器と**その役割**を明記

### リズム / プロダクション設計

```markdown
### リズム / プロダクション設計
- **基調リズム**: 16ビート + トラップ的なハイハット刻み
- **ハイハット**: Verse で細かいロール、サビ前で加速
- **ベースタイプ**: 808ベース（スライド多用、Trap由来）
- **影響元**: J-Pop × K-Pop × Trap × Hyper Pop
- **特徴的手法**: ボーカルチョップ（Drop）、サイドチェイン（サビ）
```

→ **この情報がSuno出力の精度を決める**

---

## 分析のコツ

1. **「なぜ」を常に問う** - 単なる観察ではなく設計意図を推測
2. **再利用可能な形で記述** - 他の曲にも適用できる洞察
3. **対比で特徴を際立たせる** - 「〜ではなく〜」という形式
4. **感情のアークを意識** - 曲全体で感情がどう動くか
